1. Se puede usar git blame para saber quién ha entrado código problemático, entender el contexto de los cambios hechos, y para saber a quién preguntar sobre el funcionamiento de una parte de código.

2. Es mala idea usar git blame para encontrar culpables porque ignora que los errores son colectivos, no la culpa de un compañero.

3. 
    - hacer commits pequeños con mensajes descriptivos para rastrear cambios específicos.
    - mantener un formato consistente en el código para evitar ruido y confusión.
